Feign

仅需再接口使用注解方式来配置它，即可完成对服务提供方的接口绑定。

Hystrix

用于处理分布式系统的延迟和容错的开源框架。在超时、异常等，Hystrix能够保证在一个依赖出问题的情况，避免级联故障，提高分布式弹性。

当服务发生故障，向调用方返回一个服务预期的备选响应(FallBack)，保证消费者的线程不会长时间，不必要的占用，避免雪崩。

能做到什么

  • 服务熔断
    • 避免雪崩:多级服务调用，如果一个服务响应时间过长或不可用，会导致占用越来越多的系统资源，进而引起系统崩溃。
    • 当扇出链路出现服务响应时间过长或不可用，会进行服务降级进而熔断节点调用，快速返回错误的响应信息
  • 服务降级
  • 服务限流
  • 接近事实的监控

如何使用服务熔断

服务熔断是服务端动作。

某个服务超时或异常，则会引起熔断。

  1. 引入依赖

  2. 在启动类上开启熔断@EnableCircuitBreaker

  3. 编写降级方法，并且在使用的方法上加注解@HystrixCommand(fallbackMethod = "hystrixGet")

什么是服务降级，如何使用

关闭部分服务，让出资源给需求更高更大的服务，更高效使用资源，保证服务的更高可用

服务降级是在客户端做的。

  1. 创建一个降级服务在feign接口处

  2. 在feign接口处加上注解处加上fallbackFactory，表明降级服务

    @FeignClient(value = “EUREKA-PROVIDER-DEPT”,fallbackFactory = DeptClientFallBackService.class)

  3. 在客户端(服务调用端)配置开启服务降级

踩坑注意：fallbackFactory需要注意引入的包是import feign.hystrix.FallbackFactory

Zuul

Zuul主要包含路由和过滤。

  • 路由主要做的就是实现外部访问统一入口，将外部请求转发到具体服务实例。
  • 过滤则是对请求的处理过程进行干预，实现请求校验

Zuul会注册到Eureka，再获取其他微服务，后续访问也是通过Zuul跳转所以相当于：代理+路由+过滤

Zuul可以做什么?

  1. 身份验证和安全性：识别每个资源的身份要求，拒绝不满足的请求
  2. 见解和监控：在边缘跟踪有意义的数据和统计信息。
  3. 动态路由：根据需要将请求动态路由到不同的后端集群
  4. 压力测试：主键增加集群的流量以评估性能
  5. 减载：为没中类型的请求分配容量，并求其超出限制的请求
  6. 静态响应处理：直接在边缘构建一些响应，而不是将其转发到内部集群
  7. 多区域弹性：在AWS区域之间路由请求

Spring Cloud Config分布式配置

spring cloud Config是微服务架构中为各个不同微服务应用的所有环节提供一个中心化的外部配置

它包含客户端(各个微服务)、服务端(分布式配置中心).

Spring Cloud Config能做什么

  1. 集中管理配置文件
  2. 不同环境，不同配置，动态的配置更新，如/dev 、/prod 、/test
  3. 运行期间动态调整配置，无需每个服务部署的机器上编写配置文件，服务会向配置中心统一拉取配置信息
  4. 配置变动，服务无需重启，即可应用新的配置
  5. 配置信息以Rest方式暴露
